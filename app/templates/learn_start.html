Step 1 — Authorization Request

Your app is about to redirect you to {{ provider_display }}. Here is the URL it constructed:

GET {{ authorize_url }}

Query parameters:

{% for key, value in params.items() %} {% endfor %}
{{ key }} {{ value }}
What do these parameters mean?

client_id identifies your application. You got this when you registered your OAuth app with {{ provider_display }}.

redirect_uri is where {{ provider_display }} sends the user back after they approve. It must match exactly what you registered.

scope defines what permissions you are requesting. The user will see these before approving.

state is a random token your app generates. When {{ provider_display }} redirects back, it includes this same value. Your app checks that it matches — this prevents CSRF attacks.

response_type=code means you are using the Authorization Code flow, the most secure option for server-side apps.

What happens next: You will be redirected to {{ provider_display }}. They will ask you to authorize this app. After you click "Authorize", {{ provider_display }} will redirect you back here with an authorization code.
